Appendix B  ·  Methodology

Clustering Analysis Methodology

An overview of the approach used to identify the four applicant archetypes in the Cluster Viz, including dataset scope, features used, and preprocessing steps.

Section 1

Overview

The clustering analysis characterizes applicants by the full arc of their journey — from application through early enrollment retention. The goal is descriptive: what distinct types of applicants does NHA see, and how do their journeys differ?

The pipeline applies standard scaling and one-hot encoding to prepare the data, reduces dimensions using Principal Component Analysis (PCA), and then applies K-Means clustering. The number of clusters (k = 4) was determined using the elbow method, which evaluates within-cluster variance across different values of k. Four clusters produced the clearest inflection point and the most interpretable archetypes.

Framing note All funnel stage variables — including early retention indicators like whether a student started in the first week — are treated as terminal journey outcomes rather than predictive leakage. The intent is full journey characterization, not pre-enrollment prediction.
Section 2

Dataset

The dataset contains 227,641 applicant records spanning the 2023–2025 academic years. Each record represents a single application instance and includes fields for school, grade level, academic year, lottery outcome, registration, enrollment, and early attendance indicators.

Records were filtered to exclude entries missing a valid applicant identifier or school assignment. Grade-level labels were normalized to a canonical ordering (Pre-K, K, 1–12, Unknown) for consistent filtering behavior across visualizations.

Section 3

Preprocessing & Model

The following binary features were used as clustering inputs: accepted at lottery, accepted (any path), waitlisted, registered, enrolled, and attending. Continuous features (e.g., application timing in days before school start) were also included after standard scaling.

One-hot encoding was applied to categorical features. Principal Component Analysis (PCA) was then applied to reduce the feature space to 2 components for visualization purposes while preserving the majority of variance. K-Means was then applied with k = 4, using 10 independent initializations to reduce sensitivity to starting conditions.

Section 4

Cluster Profiles

The four clusters correspond to the following applicant archetypes:

Cluster 1 — Ghost Acceptee  ·  43.0% of applicants (n = 97,566)
Cluster 2 — Committed Enrollee  ·  22.7% of applicants (n = 51,389)
Cluster 3 — Never Accepted  ·  20.5% of applicants (n = 46,435)
Cluster 4 — Partial Converter  ·  14.2% of applicants (n = 32,251)

Full characterizations of each archetype — including funnel timelines, application timing distributions, and sibling enrollment comparisons — are available in the Cluster Viz.

Data Access Statement — Data supporting this analysis cannot be made available due to client confidentiality and NDA restrictions. Code and methodological details are provided to enable replication with comparable data.
