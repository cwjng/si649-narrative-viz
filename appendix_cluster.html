Appendix B  ·  Methodology

Clustering Analysis Methodology

An overview of the approach used to identify the four applicant archetypes in the Cluster Viz, including dataset scope, features used, and preprocessing steps.

Section 1

Overview

The clustering analysis characterizes applicants by the full arc of their journey — from application through early enrollment retention. The goal is descriptive: what distinct types of applicants does NHA see, and how do their journeys differ?

The pipeline applies standard scaling and one-hot encoding to prepare the data, reduces dimensions using Principal Component Analysis (PCA), and then applies K-Means clustering. The number of clusters (k = 4) was determined using the elbow method, which evaluates within-cluster variance across different values of k. Four clusters produced the clearest inflection point and the most interpretable archetypes.

Framing note All funnel stage variables — including early retention indicators like whether a student started in the first week — are treated as terminal journey outcomes rather than predictive leakage. The intent is full journey characterization, not pre-enrollment prediction.
Section 2

Dataset

The dataset contains 227,641 applicant records spanning the 2023–2025 academic years. Features fall into three broad categories:

  • Applicant demographics Household composition, sibling enrollment status, distance from school, and homelessness indicator
  • School-level information School type and urban-centric locale designation
  • Application metadata Application timing relative to school start, acceptance and registration timing, grade applied for, lottery outcomes, and enrollment funnel stage flags (applied, accepted, registered, enrolled, attending in first week, still enrolled at day 7)

Together these features describe both who the applicant is and how they moved through (or exited) the enrollment process.

Section 3

Preprocessing

Before clustering, the data was prepared through the following steps:

  • Missing value imputation for binary flags Missing values in funnel stage indicators (e.g., accepted, registered, enrolled) were filled with 0, since absence of a record means the stage was not reached.
  • Sentinel imputation for timing variables Acceptance and registration timing variables (days before school start) were imputed with a sentinel value of 999 rather than 0. This is because negative values are meaningful in context — they indicate an applicant accepted or registered after school started. Missingness indicator flags were also created to preserve the signal that a timing record is absent entirely.
  • One-hot encoding for categorical variables Nominal categorical variables (grade level, school type, locale designation) were one-hot encoded to convert them to numeric form without imposing an artificial ordering.
  • Standard scaling for numeric variables All numeric features were standardized to zero mean and unit variance, preventing high-range continuous variables (e.g., days before school start) from dominating the clustering over binary indicators.

After preprocessing, PCA reduced the feature space to 2 components for clustering and visualization. K-Means was then applied with k = 4, using 10 independent initializations to reduce sensitivity to starting conditions.

Section 4

Cluster Profiles

The four clusters correspond to the following applicant archetypes:

Cluster 1 — Ghost Acceptee  ·  43.0% of applicants (n = 97,566)
Cluster 2 — Committed Enrollee  ·  22.7% of applicants (n = 51,389)
Cluster 3 — Never Accepted  ·  20.5% of applicants (n = 46,435)
Cluster 4 — Partial Converter  ·  14.2% of applicants (n = 32,251)

Full characterizations of each archetype — including funnel timelines, application timing distributions, and sibling enrollment comparisons — are available in the Cluster Viz.
